# DED-118 Workpad

## Summary

Add `toolist weclaw status`, `toolist weclaw bind`, and `toolist weclaw relay` to the CLI. The commands use the local WeClaw bridge at `http://127.0.0.1:18011` by default and authenticated Gateway endpoints for binding completion and delivery claim/ack.

## Intended Change

- Add a local WeClaw HTTP client for `/health` and `/api/send`.
- Add command modules for status, bind, and relay.
- Wire a `weclaw` command group into `src/cli.ts` with help and argument parsing.
- Add integration tests covering help, dispatch, local send success, failed local send ack, and JSON stdout behavior.
- Rework the Gateway contract mapping after review:
  - `bind` sends `targetUserId` / `targetLabel` and parses `data.binding`.
  - `relay` claim sends `relayId` and accepts delivery `to`.
  - failed ack sends `errorMessage`.
- Rework the opencode review findings:
  - `weclaw status` rejects hosted API flags instead of silently ignoring them.
  - root help uses `toollist weclaw status` consistently in its discovery block.
